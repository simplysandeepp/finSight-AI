PHASE 0

Offline Data & Training Pipeline

generator.py
Synthetic Data Generator
40 companies × 20 quarters = 800 rows GBM or AR(1) revenue simulation Bull / Neutral / Bear scenarios (40/40/20%) Mismatch flags, restatement rate 10 transcript templates w/ sentiment
→ out/synthetic_financials.csv
→ out/sample_transcripts.jsonl
feature_store.py
Feature Engineering
Lag features: revenue 1Q/2Q/4Q, EBITDA 1Q Rolling 4Q mean & std (revenue, EBITDA margin) YoY & QoQ growth rates One-hot encode scenario column Temporal split 70 / 15 / 15 by date Drop NaN rows from lag windows
← synthetic_financials.csv
→ out/features_v1.pkl
→ out/feature_manifest.json
financial_model.py train()
XGBoost Model Training
2 targets: revenue, EBITDA 3 quantiles: α = 0.05, 0.50, 0.95 = 6 XGBoost models trained Objective: reg:quantileerror 100 estimators, max_depth=5, hist method Val set used for early eval
← features_v1.pkl (train+val splits)
→ out/financial_model.pkl

PHASE 2

Orchestrator — Data Routing & Source Selection

orchestrate.py
Orchestrator — Master Controller
① Data Source Decision
If company_id is a real ticker (no "COMP_" prefix) Try yfinance → 8 quarters of quarterly financials compute_features() for lag/rolling features Fallback → Alpha Vantage news sentiment Else → load from features_v1.pkl (synthetic)
② Transcript & Peers
Synthetic: match company_id + quarter in .jsonl Live: static placeholder text (gap!) Peers: PEER_MAP[ticker] → yfinance fetch Peer tickers: AAPL→MSFT/GOOGL/META etc. No peer data for synthetic COMP_XXX IDs
③ Parallel Dispatch
asyncio.gather() — all 4 agents fire simultaneously asyncio.wait_for() — 10s hard timeout per agent Failed/timed-out agents added to degraded[] Minimum 2 agents must succeed (else 503) Confidence penalised: × 0.9^(num_degraded)
⚠ Degraded Mode: If any agent times out or throws, it returns a fallback output (confidence=0.5) and is added to degraded_agents[]. The orchestrator continues with remaining agents. If <2 agents succeed, the entire request fails with InsufficientDataError.
PHASE 3

Parallel Agent Execution

asyncio.gather() — All 4 agents run simultaneously in parallel
Agent 1
Transcript NLP Agent
🦙 Groq llama-3.3-70b ✦ Gemini 2.0 fallback
IN: transcript_text, company_id, quarter
Extract 5 key driver sentences Sentiment score [-1, 1] Mgmt confidence [0, 1] Top 5 topics with scores All numeric facts (rev, EBITDA, guidance) Flag optimistic language + bad numbers 10s asyncio timeout JSON cleanup + regex fallback parse
OUT: drivers, sentiment, topics, numeric_facts, confidence
DEGRADED: confidence=0.5, empty lists
Agent 2
Financial Model Agent
🌲 XGBoost (no LLM)
IN: company_id, as_of_date, features dict
Loads financial_model.pkl (6 models) Predicts revenue p05/p50/p95 Predicts EBITDA p05/p50/p95 Enforces monotonicity: p05 ≤ p50 ≤ p95 Confidence = 1 - (p95-p05)/(2×p50) Feature importance from p50 rev model (gain) Top 5 features returned
OUT: revenue_forecast {p05,p50,p95}, ebitda_forecast, feature_importances, confidence
DEGRADED: RuntimeError → caught by orchestrator
Agent 3
News & Macro Agent
🦙 Groq llama-3.3-70b ✦ Gemini 2.0 fallback
IN: headlines[], macro_indicators {gdp_growth, interest_rate}
Analyze news headlines + macro data Returns macro_sentiment [-1,1] Returns sector_outlook string Returns risk_factors list 10s asyncio timeout Headlines: live via Alpha Vantage or synthetic
OUT: macro_sentiment, sector_outlook, risk_factors, confidence
DEGRADED: sentiment=0, empty risk_factors
Agent 4
Competitor Agent
🦙 Groq llama-3.3-70b ✦ Gemini 2.0 fallback
IN: peer_financials[], market_share_signals
Benchmarks vs. peer companies Computes revenue_delta & margin_delta per peer Returns relative_position_score [-1, 1] Handles peer alias mapping robustly Peers empty for synthetic COMP_XXX IDs 10s asyncio timeout
OUT: relative_position_score, peer_benchmarks[], confidence
DEGRADED: score=0, empty benchmarks
All LLM agents call via LLMClient singleton
llm_client.py — Singleton
LLM Rotation & Fallback Client
Primary: Groq (Key Rotation)
Loads all GROQ_API_KEY* env vars Round-robin rotation on 429 RateLimitError AsyncGroq client, temp=0 Model: llama-3.3-70b-versatile
Fallback: Gemini
Triggered if ALL Groq keys fail GEMINI_API_KEY from .env asyncio.to_thread (sync→async wrapper) Model: gemini-2.0-flash, temp=0
Error Path
Both fail → LLMUnavailableError raised Each agent catches → degraded output Orchestrator logs degraded_agents[]
All agent outputs merge → Ensembler
